# EM플러스 학원 학생 진도 관리 시스템
## 앱 제안서

---

## 1. 프로젝트 개요

### 1.1 프로젝트명
**EM플러스 학원 – 학생 진도 현황 대시보드**

### 1.2 목적 및 배경
학원 운영에서 학생별 학습 진도, 성적, 코멘트를 체계적으로 관리하고, 원장·선생님·학생 간 실시간 소통 채널을 구축하여 학원 운영 효율을 극대화하는 것을 목표로 합니다.

기존 학원의 문제점:
- 학생별 진도 파악이 수기/엑셀 기반으로 비효율적
- 선생님 간 학생 정보 공유가 구두에 의존
- 학부모 대상 진행 상황 전달이 체계적이지 않음
- 시험 성적 추이 분석이 어려움

### 1.3 기대 효과
| 항목 | 효과 |
|------|------|
| 진도 관리 효율화 | 학생별 진행률을 실시간 시각화하여 즉시 파악 가능 |
| 성적 추이 분석 | 시험별 과목별 점수 변화를 차트로 한눈에 확인 |
| 소통 강화 | 원장-선생님 내부 메시지, 학생별 코멘트 시스템으로 정보 단절 해소 |
| 의사결정 지원 | 대시보드 통계/차트로 데이터 기반 학원 운영 판단 가능 |
| 접근성 향상 | 웹 기반으로 PC·모바일 어디서나 접근 가능 |

| 역할 | 한글명 | 설명 |
|------|--------|------|
| **Director** | 원장 | 학원 전체 관리자. 모든 데이터 접근 및 관리 권한 보유 |
| **Teacher** | 선생님 | 담당 학생에 대한 학습 관리 권한 보유 |
| **Student** | 학생 | 본인 정보 조회 및 제한적 기능 사용 |

### 2.2 역할별 접근 권한 매트릭스

|------|:----:|:------:|:----:|
| 대시보드 | ✅ 전체 | ✅ 담당 학생 | ✅ 본인만 |
| 학생 관리 | ✅ 전체 CRUD | ✅ 담당 학생 조회 | ✅ 본인 조회만 |
| 학습 계획 | ✅ | ✅ | ❌ |
| 진도 현황 | ✅ | ✅ | ❌ |
| 코멘트 | ✅ | ✅ | ❌ |
| 성적 관리 | ✅ | ✅ | ✅ 본인만 |
| 선생님 관리 | ✅ | ❌ | ❌ |
| 내부 소통 | ✅ | ✅ | ✅ |

### 2.3 가입 및 승인 프로세스

```
[회원가입] → approved: false → [원장 승인] → 로그인 가능
                                └─ [원장 거절] → 계정 삭제
```

- 회원가입 시 역할(원장/선생님/학생) 선택
- 학생 가입 시 학교·학년·반·연락처 추가 입력 → `students` 테이블에 자동 등록
- 모든 신규 계정은 원장 승인 후에만 로그인 가능

### 3.3 데이터 동기화 패턴

```
[사용자 조작]
    ↓
[메모리 캐시 업데이트] (즉시 응답, 낙관적 업데이트)
    ↓
[Supabase API 호출] (비동기)
    ├─ 성공 → 완료
    └─ 실패 → 캐시 롤백 (backup 복원)
```

- 앱 시작 시 Supabase에서 전체 데이터를 메모리 캐시로 로드
- CUD 작업 시 로컬 캐시 먼저 업데이트 → UI 즉시 반영 → 서버 동기화
- camelCase(JS) ↔ snake_case(PostgreSQL) 자동 변환

## 4. 화면별 상세 기능

### 4.1 로그인 / 회원가입 화면

**로그인**
- 아이디 + 비밀번호 입력
- 미승인 계정 접속 시 "승인 대기 중" 안내
- 로그인 성공 시 `localStorage`에 세션 저장

**회원가입**
- 3개 탭: 원장 / 선생님 / 학생
- 공통 입력: 이름, 아이디, 비밀번호(4자 이상), 비밀번호 확인
- 학생 추가 입력: 학교명, 학년(중1~고3), 반, 전화번호, 학부모 전화번호

---

### 4.2 대시보드

학원 전체(또는 담당 학생) 현황을 한눈에 파악하는 메인 화면

**통계 카드** (4종)
| 카드 | 내용 | 아이콘 색상 |
|------|------|------------|
| 학생 수 | 총 관리 학생 수 (역할별 필터 적용) | 파란색 |
| 학습 계획 | 활성 상태 학습 계획 수 | 녹색 |
| 평균 진행률 | 전체 활성 계획의 평균 완료율(%) | 노란색 |
| 코멘트 수 | 전체 코멘트 수 | 보라색 |

**차트** (4종)
1. **학습 계획 현황 도넛**: 완료 / 진행중 / 대기 비율
2. **학생 진도 분포 바**: 0~25% / 26~50% / 51~75% / 76~100% 구간별 학생 수
3. **과목별 평균 진행률 바**: 수학·영어·국어 등 과목별 평균 진행률
4. **최근 활동 타임라인**: 최신 코멘트 5건

**학생별 진도 요약 테이블**
- 학생명, 학교/학년, 반, 진행 계획 수, 평균 진행률 (프로그레스 바)

---

### 4.3 학생 관리

**학생 목록**
- 학년 필터 (중1~고3)
- 반 필터 (동적 생성)
- 글로벌 검색 (이름/학교 대상)
- 학생 카드 클릭 → 상세 페이지 이동

**학생 등록/수정 폼**
| 필드 | 타입 | 비고 |
|------|------|------|
| 이름* | 텍스트 | 필수 |
| 학교 | 텍스트 | |
| 학년 | 선택 | 중1~고3 |
| 반 | 텍스트 | |
| 전화번호 | 전화 | |
| 학부모명 | 텍스트 | |
| 학부모 전화 | 전화 | |
| 이전 성적 | 텍스트영역 | 자유 기술 |
| 특이사항 | 텍스트영역 | 메모 |

**학생 삭제**
- 확인 팝업 후 cascade 삭제 (관련 학습계획, 진도기록, 코멘트 전부 삭제)

---

### 4.4 학생 상세 페이지

학생 한 명에 대한 모든 정보를 통합 표시하는 페이지

**구성 요소**

1. **학생 정보 카드**
   - 아바타 (이름 이니셜), 기본 정보, 담당 선생님 목록

2. **과목별 진행률 시각화**
   - 수평 바 차트 + 레이더 차트

3. **학습 계획 카드**
   - 과목, 교재, 난이도 배지, 진행률 바, 남은 기간

4. **진도 타임라인 차트**
   - 누적 진행량 라인 + 목표선 (빨간 점선)

5. **성적 피벗 테이블**
   - 행: 과목, 열: 시험, 셀: 점수 (+추이 화살표 📈📉→)

6. **코멘트 스레드**
   - 역할별 아이콘, 시간순 정렬

---

- 각 카드: 학생명, 과목, 교재, 난이도 배지, 진행률 바, 기간

| 학생* | 선택 | 관리 학생 목록 |
| 과목* | 텍스트 | |
| 교재명 | 텍스트 | |
| 학습 방법 | 텍스트 | |
| 난이도 | 선택 | 상 / 중 / 하 |
| 계획 유형 | 선택 | 중간고사 / 기말고사 / 방학 / 수시 |
| 시작일 | 날짜 | |
| 종료일 | 날짜 | |
| 총 분량 | 숫자 | |
| 단위 | 선택 | 페이지 / 단원 / 챕터 / 문제 / 세트 / 지문 |
| 현재 진행량 | 숫자 | |
| 상태 | 선택 | 진행중 / 완료 / 일시중지 |

### 4.6 진도 현황

**빠른 진도 입력 폼**
1. 학생 선택
2. 학습 계획 선택 (활성 계획만 표시)
3. 진행량 입력 + 날짜 + 메모
4. 저장 시 학습 계획의 `completedUnits` 자동 갱신

**진행률 색상 체계**
| 진행률 | 색상 | 의미 |
|--------|------|------|
| 76~100% | 🟢 초록 | 양호 |
| 41~75% | 🔵 기본 (인디고) | 보통 |
| 21~40% | 🟡 노란 | 주의 |
| 0~20% | 🔴 빨강 | 위험 |

**최근 진도 기록 테이블**
- 날짜, 학생명, 과목, 진행량, 메모 표시

---

### 4.7 성적 관리

**시험 유형별 입력 필드**

| 항목 | 중간/기말고사 | 모의고사 |
|------|:------------:|:--------:|
| 학기 | ✅ | ❌ |
| 시험명 | ❌ | ✅ |
| 과목별 원점수 | ✅ | ✅ |
| 등급 | A+~F | 1~9등급 |
| 석차 | ✅ | ❌ |
| 표준점수 | ❌ | ✅ |
| 백분위 | ❌ | ✅ |
| 전체 석차 | ✅ | ❌ |

**3종 뷰 모드**

1. **테이블 뷰**
   - 과목별 행 전개, 8개 컬럼 정렬 지원
   - 학생 / 학기 / 시험유형 / 과목 다중 필터

2. **피벗 뷰**
   - 학생별 카드
   - 행: 과목, 열: 시험
   - 점수 추이 표시 (📈 상승 / 📉 하락 / → 유지)

3. **차트 뷰**
   - 과목별 점수 추이 라인 차트
   - 시험별 평균 비교 바 차트

**성적 입력 폼**
- 동적 과목 행 추가/삭제
- 시험유형 변경 시 입력 필드 실시간 전환

### 4.8 코멘트

**코멘트 작성**
| 필드 | 설명 |
|------|------|

**코멘트 목록**
- 학생별 / 역할별 필터
- 수신 대상 태그 표시
- 시간순 정렬

### 4.9 내부 소통 (원장 ↔ 선생님 메시지)

**메시지 게시판**
- 작성자: 원장 또는 선생님
- 관련 학생 지정 가능 (optional)
- 제목 + 본문

**핵심 기능**
| 기능 | 설명 |
|------|------|
| 📌 고정 | 중요 메시지 상단 고정/해제 |
| ✅ 읽음 확인 | 수신자별 읽은 시간 추적 |
| 🔴 미읽음 배지 | 사이드바 메뉴에 실시간 미읽음 수 표시 |
| 🔍 필터 | 작성자 역할별, 관련 학생별 |

**수신자 체계**
- 원장 작성 → 모든 선생님에게 전달
- 선생님 작성 → 원장에게 전달

### 4.10 선생님 관리 (원장 전용)

**기능 목록**
- 가입 대기 목록 (승인/거절)

## 5. 데이터베이스 설계

### 5.1 ERD (Entity Relationship)

```
┌──────────────┐     1:N     ┌──────────────┐

│              │ student_ids │              │
└──────────────┘             └──────┬───────┘
                                   │
                    ┌──────────────┼──────────────┐
                    │              │              │
                    ▼ 1:N          ▼ 1:N          ▼ 1:N
             ┌──────────┐  ┌──────────┐  ┌──────────┐
             │  plans   │  │  grades  │  │ messages │
             │(학습계획) │  │ (성적)    │  │ (메시지)  │

         │               │
```

## 7. 핵심 기술 특징

### 7.1 SPA (Single Page Application)
- 프레임워크 없이 Vanilla JS로 구현
- `App.navigate()` 메서드로 뷰 전환, `innerHTML` 기반 DOM 교체
- 이벤트 위임 패턴: `data-action` 속성 기반 클릭 핸들링

### 7.2 낙관적 업데이트 (Optimistic Update)
- 로컬 캐시 먼저 업데이트 → UI 즉시 반영 → 서버 비동기 동기화
- 서버 실패 시 캐시 롤백으로 데이터 일관성 보장

### 7.3 XSS 방어
- `escapeHtml()` 유틸리티로 사용자 입력 이스케이핑
- `textContent` / `innerHTML` 패턴을 통한 안전한 DOM 조작

### 7.4 차트 관리
- Chart.js 인스턴스를 맵으로 추적
- 뷰 전환 시 `destroyAll()`로 메모리 누수 방지
